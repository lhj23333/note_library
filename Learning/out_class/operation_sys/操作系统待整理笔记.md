
- 程序运行的实现：
	- 运行处理器分配运行内存  ->  进程
	- 读取代码并执行

- 操作系统处理有问题的应用程序的一种方法：崩溃
	- 其实质是使应用程序程序所在的进程分段错误，从而导致进程崩溃，停止程序

- 应用程序不可以修改操作系统的内存（即进程不能反向影响操作系统对其所分配的运行内存），否则操作系统会崩溃。同时应用程序之间的数据不应该直接相通共享（修改），以保护数据的私有性。

- 为避免操作系统体系中，各个部分可以稳定运行，所以需进行
	- 不同进程之间的隔离
	- 操作系统与进程之间的隔离

- 操作系统体系的理解
	- 操作系统是一种服务，是运行处理器进行系统处理的软件抽象
	- 运行处理器通过控制运行存储器来实现不同进程的分配
	- 同时单条进程之上可能存在多个应用程序
	- 而多个应用程序之间可以通过相关的协议服务来实现数据的通信

- 不同进程之间数据通信的实现：
	- Socket：套接字通信
		- 通过网络接口实现同一台机器或者不同机器之间的不同进程相互进行通信
	- 同一台机器之间的数据通信：
		- 可通过共享内存（即共享硬件存储器）来实现数据通信
		- 应用程序可以将一段数据写入存储中的本地文件系统，甚至是全局文件系统，其他应用程序进行读取从而实现数据通信

- 调整进程即调整 CPU
	- CPU 是进程执行的硬件核心，只有进程获得 CPU 时间片后才能运行代码
		- 一个 CPU 在任意时刻只能执行一个进程
	- 进程任务是通过 CPU 的指令周期（取指令、解码、执行、写回）完成的
	- 操作系统通过调度算法来实现 CPU 时间的分配

- RISC-V 汇编语言:
	- 一种开放指令集架构
		- 不需要支付任何许可证就可以使用它来构建流程并围绕它构建处理器

- 操作系统的作用：
	- 为应用程序提供简洁、易于使用的物理资源抽象，使得应用程序开发人员可以更轻松地开发此应用程序
	- 通过提供清晰的抽象，操作系统可为应用程序提供保护和隔离、以及跨多个应用程序的资源共享
	- 为应用程序提供协议服务，使得应用程序之间可以进行数据通信

- 应用程序与常见服务之间的交互，可通过 API 应用程序接口来实现
	- 例如，通过系统库公开
		- 当你想要访问文件时，你可以通过对应的 API 来实现读取访问并查找相同的内容
		- 当你相进行通信发送和接收数据时，你可以通过对应的 API 来实现

- 随着时代的发展，在前期硬件资源能力与软件程序等的发展速度旗鼓相当，都呈现快速发展趋势，到了近代，硬件资源的发展速度受限趋于缓慢（不遵循摩尔定律），软件程序复杂度保持指数上升（大模型等），所以对硬件资源管理分配的重要性越发凸显，即操作系统以及硬件架构诞生的意义。
